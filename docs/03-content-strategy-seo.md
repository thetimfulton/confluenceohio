## 2. SEO Keyword Strategy

### 2.1 Keyword Landscape Assessment

The SEO landscape for this campaign is **relatively uncrowded**, offering significant first-mover advantage. The primary renaming keywords have low-to-medium competition, and the long-tail historical and geographic keywords are largely uncontested by authoritative sites. The main challenge is that generic "Confluence" searches are dominated by Atlassian Confluence software documentation.

**Competition levels:**
- **Low competition:** Direct renaming keywords, city-renaming process terms, Joseph Foos / founding history
- **Medium competition:** Long-tail history and geography keywords, Indigenous Peoples' Day content, Christopher Columbus legacy
- **High competition:** Generic "Confluence" (software noise), broad Christopher Columbus queries (Wikipedia dominates), Ohio history (Ohio History Connection)

### 2.5 Paid Search Strategy

**Critical finding: Google Ad Grants is NOT available to 501(c)(4) organizations.** Google Ad Grants requires 501(c)(3) status. As a 501(c)(4) civic league, Confluence Ohio is permanently ineligible unless a separate 501(c)(3) educational affiliate is established.

**Approach: Organic-first with $500/month paid media.**

The campaign will rely heavily on organic search and earned media, supplemented by a modest $500/month paid budget. A 501(c)(3) educational affiliate is not planned at this time but remains a future option if the campaign scales and the $10K/month Ad Grants becomes worth the legal overhead.

**$500/month paid media allocation:**
1. **Google Ads (~$300/month):** Target high-intent keywords ("rename Columbus Ohio," "Confluence Ohio petition"). Low competition means CPCs of $0.50–$2.00, yielding 150–600 clicks/month. Focus on search ads only — no display.
2. **Meta Ads (~$200/month):** Geographic targeting to Columbus metro (Franklin County + adjacent counties). Retarget site visitors who didn't sign. Promote milestone moments and key blog posts. Reserve budget for boosted posts during signature pushes.

**Organic-first strategy priorities:**
- SEO content production (Pillars 1–3) is the primary traffic driver — target informational keywords with low competition and high intent
- Local media earned coverage (Columbus Dispatch, WOSU, Columbus Underground) for backlinks and authority
- Social sharing virality via the referral system and post-signature share flow
- Email list growth via petition auto-subscribe and blog signups — email drives return visits and re-engagement

## 3. Schema Markup Plan

### 3.1 Organization Schema (Site-Wide)

Applied via `<script type="application/ld+json">` in the root layout. Present on every page.

```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "Confluence Ohio",
  "url": "https://confluenceohio.org",
  "logo": "https://confluenceohio.org/images/logo.png",
  "description": "A 501(c)(4) civic organization building a movement to rename Columbus, Ohio to Confluence, Ohio.",
  "nonprofitStatus": "Nonprofit501c4",
  "foundingDate": "2026",
  "address": {
    "@type": "PostalAddress",
    "addressLocality": "Columbus",
    "addressRegion": "OH",
    "addressCountry": "US"
  },
  "contactPoint": {
    "@type": "ContactPoint",
    "email": "info@confluenceohio.org",
    "contactType": "customer service"
  },
  "sameAs": [
    "https://www.facebook.com/confluenceohio",
    "https://twitter.com/confluenceohio",
    "https://www.instagram.com/confluenceohio"
  ]
}
```

**Note:** Schema.org supports `Nonprofit501c4` as a valid `nonprofitStatus` value. This is the correct designation for a civic league under IRC 501(c)(4).

### 3.2 BreadcrumbList Schema (All Interior Pages)

Applied dynamically based on URL structure. Example for `/the-case/history`:

```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [

```

**Implementation:** Generate breadcrumbs automatically from the Next.js route segments. Create a shared `<BreadcrumbJsonLd>` component that reads the current route and outputs the correct schema.

### 3.3 Article / BlogPosting Schema (Blog Posts and Voice Stories)

Applied to all `/blog/[slug]` and `/voices/[slug]` pages.

```json
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": "{{title}}",
  "description": "{{meta description}}",
  "datePublished": "{{ISO 8601 date}}",
  "dateModified": "{{ISO 8601 date}}",
  "author": {
    "@type": "Person",
    "name": "{{author name, default 'Confluence Ohio'}}"
  },
  "publisher": {
    "@type": "Organization",
    "name": "Confluence Ohio",
    "logo": {
      "@type": "ImageObject",
      "url": "https://confluenceohio.org/images/logo.png"
    }
  },
  "image": "{{OG image URL}}",
  "mainEntityOfPage": {
    "@type": "WebPage",
    "@id": "{{canonical URL}}"
  }
}
```

For `/voices/[slug]`, use `"@type": "Article"` instead of `"BlogPosting"`. Include the author's `"name"` from the voice submission and omit `"dateModified"` if not applicable.

### 3.4 FAQPage Schema — Approach Update

**Important:** As of 2023, Google significantly restricted FAQPage rich results. They now only appear for well-known government and health websites. For a civic campaign site, FAQPage schema will **not produce rich snippets** in Google.

**Recommendation:** Still implement FAQPage schema on `/faq` for semantic correctness and potential future re-expansion, but **do not rely on it for SERP visibility.** Instead, optimize the FAQ page for standard organic ranking on question-based keywords.

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "What is Confluence Ohio?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Confluence Ohio is a 501(c)(4) civic organization..."
      }
    }
  ]
}
```

### 4.3 Dynamic OG Images

Three categories of pages require dynamically generated OG images:

**1. Blog posts (`/blog/[slug]`):**
- Auto-generated from post title + author + date
- Template: campaign-branded background, post title in headline font, author name, Confluence Ohio logo
- Generated at build time (SSG) or on-demand (ISR) via `next/og` (ImageResponse API)

**2. Voice stories (`/voices/[slug]`):**
- Quote excerpt (first ~100 characters) + author name + position badge (Support/Oppose/Undecided)
- Color-coded by position: green (support), red (oppose), amber (undecided)
- Generated on-demand via `next/og`

**3. Petition milestones (periodic update for `/sign`):**
- "Join [X] Ohioans Who've Signed" with current count
- Updated via a scheduled job (e.g., every 6 hours) — not real-time, since OG images are cached aggressively by social platforms
- Stored as a static file that gets overwritten: `/public/og/sign.png`

**Technical implementation with `next/og`:**

```typescript
// app/api/og/blog/route.tsx
import { ImageResponse } from 'next/og';

export async function GET(request: Request) {
  const { searchParams } = new URL(request.url);
  const title = searchParams.get('title') ?? 'Confluence Ohio';
  const author = searchParams.get('author') ?? 'Confluence Ohio';

  return new ImageResponse(
    (
      <div style={{
        display: 'flex',
        flexDirection: 'column',
        width: '100%',
        height: '100%',
        backgroundColor: '#1a365d', // campaign brand color
        padding: '60px',
        justifyContent: 'space-between',
      }}>
        <div style={{ fontSize: 48, color: '#ffffff', fontWeight: 700 }}>
          {title}
        </div>
        <div style={{ display: 'flex', justifyContent: 'space-between', alignItems: 'flex-end' }}>
          <div style={{ fontSize: 24, color: '#a0aec0' }}>
            By {author}
          </div>
          <div style={{ fontSize: 20, color: '#a0aec0' }}>
            confluenceohio.org
          </div>
        </div>
      </div>
    ),
    { width: 1200, height: 630 }
  );
}
```

**Constraints for `next/og` (Satori engine):**
- Flexbox-only layout (no CSS Grid)
- Supports TTF/OTF fonts (load campaign fonts explicitly)
- 500KB maximum bundle size for the OG route
- Use Edge Runtime for low latency

**Key gotchas:**

- X falls back to `og:image` if `twitter:image` is absent — but X's preferred aspect ratio is 2:1 (1200×600) while Facebook prefers 1.91:1 (1200×628). Use 1200×630px as the compromise.
- WhatsApp uses `og:image` — no separate tags needed, but WhatsApp will not render images larger than ~300KB.
- iMessage uses `og:image` and `og:title` — generally works without extra configuration.

## 5. Content Production Workflow

### 5.1 Blog Posts and Campaign Updates

```
IDEA → DRAFT → REVIEW → PUBLISH → DISTRIBUTE
```

| Stage | Owner | Tool/Location | Duration |
|-------|-------|---------------|----------|
| **Idea** | Anyone on team | Content calendar (Google Sheets — shared editorial calendar with columns for title, pillar, keyword target, author, status, publish date) | Ongoing |
| **Draft** | Author (individual or "Confluence Ohio") | Google Docs (linked from Google Sheets calendar) or direct in CMS | 1–3 days |
| **Review** | Campaign editor (Tim or designated editor) | Commenting in draft tool | 1–2 days |
| **Fact-check** | Author + editor | Verify all claims against Artifact 01 historical-research.md; web search for any new claims | Same as review |
| **SEO check** | Author | Verify: target keyword in title, first paragraph, one H2, and meta description. Internal links to at least 2 other site pages. Alt text on all images. | Pre-publish |
| **Publish** | Editor (has CMS publish access) | Supabase blog_posts table or MDX commit + deploy | Same day as approval |
| **Distribute** | Social media manager | Post to Facebook, X, Instagram. Send to email list if significant. Add to GBP posts. | Within 2 hours of publish |

**Publishing cadence:**
- Launch week: 3–4 posts (launch essay, first signature milestone, "how the process works" explainer, first community voice roundup)
- Ongoing: 2 posts/week minimum (1 content pillar article + 1 campaign update)
- Milestone moments: Immediate blog post + social blast + email to full list

### 5.2 Community Voice Submissions

```
SUBMIT → AI MODERATION → [HUMAN REVIEW if needed] → PUBLISH → NOTIFY
```

| Stage | Handler | Tool | Duration |
|-------|---------|------|----------|
| **Submit** | Visitor | `/voices/share` form | — |
| **AI moderation** | Automated (Claude API via Inngest) | Checks respectfulness, PII, spam, off-topic | <1 minute |
| **Auto-approve** | System | High-confidence respectful content → published immediately | <1 minute |
| **Needs review** | Human moderator | Admin dashboard queue with AI reasoning | Within 48 hours |
| **Auto-reject** | System | Clear violations → rejection email to submitter | <1 minute |
| **Publish** | System or moderator | Creates `/voices/[slug]` page, adds to voices grid | Immediate |
| **Notify** | System (Brevo) | Confirmation email to author: "Your perspective is live" with link | Immediate post-publish |

**Moderation principles:**
- We publish perspectives we disagree with. Moderation is for respectfulness, not agreement.
- No hate speech, personal attacks, doxxing, or spam.
- Borderline cases default to publish (we erred on the side of inclusion).
- All moderation decisions are logged for transparency and appeal.
